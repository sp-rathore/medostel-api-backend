## Paths

### GET /users/search

Search for a user by email address or mobile number.

**Operation ID:** searchUser

```yaml
tags:
  - Users
summary: Search user by email or mobile
description: |
  Retrieves user information by searching with either email address or mobile number.
  Returns user details if found, otherwise returns existsFlag=false.

  At least one query parameter (emailId or mobileNumber) must be provided.
operationId: searchUser
parameters:
  - name: emailId
    in: query
    description: User email address (max 255 chars)
    required: false
    schema:
      type: string
      format: email
      maxLength: 255
    example: john.doe@example.com

  - name: mobileNumber
    in: query
    description: User mobile number (10 digits, 1000000000-9999999999)
    required: false
    schema:
      type: integer
      minimum: 1000000000
      maximum: 9999999999
    example: 9876543210

responses:
  '200':
    description: Search operation successful
    content:
      application/json:
        schema:
          $ref: '#/components/schemas/UserSearchResponse'
        examples:
          userFound:
            summary: User found
            value:
              data:
                userId: USER_001
                firstName: John
                lastName: Doe
                currentRole: DOCTOR
                emailId: john.doe@example.com
                mobileNumber: 9876543210
                organisation: Apollo Hospital
                address1: 123 Medical Street
                address2: Suite 101
                stateId: MH
                stateName: Maharashtra
                districtId: null
                cityId: CITY_001
                cityName: Mumbai
                pinCode: null
                commentLog: null
                status: active
                createdDate: "2026-03-03T10:30:00"
                updatedDate: "2026-03-03T10:30:00"
              existsFlag: true
          userNotFound:
            summary: User not found
            value:
              data: null
              existsFlag: false

  '400':
    description: Bad request - validation error
    content:
      application/json:
        schema:
          $ref: '#/components/schemas/ErrorResponse'
        examples:
          missingParameters:
            summary: Missing both parameters
            value:
              detail: At least one of emailId or mobileNumber must be provided
          invalidEmail:
            summary: Invalid email format
            value:
              detail: Invalid email format
          invalidMobile:
            summary: Invalid mobile number
            value:
              detail: Mobile number must be 10 digits (1000000000-9999999999)

  '500':
    description: Internal server error
    content:
      application/json:
        schema:
          $ref: '#/components/schemas/ErrorResponse'
```

## Validation Rules

### Email Validation
- **Pattern:** `^[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Za-z]{2,}$`
- **Max Length:** 255 characters
- **Uniqueness:** Must be unique across all users
- **Case Handling:** Converted to lowercase
- **Validation Point:** API layer (field validator)

### Mobile Number Validation
- **Range:** 1000000000 to 9999999999 (exactly 10 digits)
- **Type:** Integer
- **Uniqueness:** Must be unique across all users
- **Validation Points:**
  - Database: CHECK constraint
  - API: Field validator with Pydantic

### Status Field
- **Valid Values:** `active`, `pending`, `deceased`, `inactive`
- **Case Handling:** Input normalized to lowercase
- **Default:** `active`
- **Validation:** Database CHECK constraint + API validation

### Role Names (8 Valid Roles)
1. `ADMIN` - System Administrator
2. `DOCTOR` - Doctor/Physician
3. `HOSPITAL` - Hospital Administrator
4. `NURSE` - Nurse
5. `PARTNER` - Sales Partner
6. `PATIENT` - Patient
7. `RECEPTION` - Reception Staff
8. `TECHNICIAN` - Lab Technician

- **Case Handling:** Input normalized to uppercase
- **Validation:** API field validator with enum check

### Name Constraints
- **Min Length:** 1 character
- **Max Length:** 50 characters
- **Required:** Both firstName and lastName must be provided in create
- **Fields:** firstName, lastName

---

## Unique Constraints

1. **emailId** - Must be unique across all users
2. **mobileNumber** - Must be unique across all users
3. **(emailId, mobileNumber)** - Composite unique constraint

## Immutable Fields

The following fields cannot be changed after creation:

| Field | Reason |
|-------|--------|
| **userId** | Primary key, used for all references |
| **createdDate** | Record creation timestamp, must remain immutable |

Attempting to update these fields will be silently ignored (no error raised).

## Example Usage

### cURL - Search User

```bash
# Search by email
curl -X GET "http://localhost:8000/api/v1/users/search?emailId=john@example.com"

# Search by mobile
curl -X GET "http://localhost:8000/api/v1/users/search?mobileNumber=9876543210"
```

### cURL - Create User

```bash
curl -X POST "http://localhost:8000/api/v1/users" \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "John",
    "lastName": "Doe",
    "currentRole": "DOCTOR",
    "emailId": "john.doe@hospital.com",
    "mobileNumber": 9876543210,
    "organisation": "Apollo Hospital",
    "status": "active"
  }'
```

### cURL - Update User

```bash
curl -X PUT "http://localhost:8000/api/v1/users/USER_001" \
  -H "Content-Type: application/json" \
  -d '{
    "firstName": "Jonathan",
    "organisation": "Max Hospital",
    "commentLog": "Updated name and organization after job transfer"
  }'
```

### Python Requests

```python
import requests

# Search user
response = requests.get(
```
